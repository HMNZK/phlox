# ADR 0088: 接続確立を QR ペアリングに一本化し、手動の認証情報入力・供給 UI を撤去する

> **このファイルの役割**: ADR 0076 の QR 採用方針を、iOS と macOS の唯一の接続経路として具体化した決定を記録する。
> **書かないもの**: 現行の画面・モジュール構成（→ architecture）や QR ペイロード契約（→ specs）。

## 文脈

[ADR 0076](0076-adopt-qr-pairing-for-mobile.md) はモバイルのペアリングに QR コードを採用し、[ADR 0074](0074-mobile-remote-control-design.md) はモバイルトークンを Mac 遠隔操作の全権として扱う脅威モデルを定めた。一方、iOS には token・host・port の手動入力経路が、macOS にはトークン文字列の表示・クリップボードコピーによる手動供給経路が残っていた。

スマートフォンから Mac の `MobileProxy` へ到達するには Tailscale が実質必須である。Tailscale IPv4 を解決できない場合、`MobileProxy` は `127.0.0.1` のみに bind し、スマートフォンから到達できない。QR ペイロードも Tailscale 到達可能な host を前提に生成される。

手動 host を想定した `explicitHost` 経路は存在したが、CompositionRoot は `MobileProxy(targetPort:)` を生成する際に `listenHost` を渡しておらず、アプリの接続経路から使用されないデッドコードだった。

## 決定

接続確立の唯一のユーザー導線を QR 読み取りにする。

- iOS の接続設定画面から token・host・port の手動入力と手動保存を撤去する。現在の接続先は読み取り専用で表示し、接続／再接続は「QRで接続」だけから行う。疎通テストは保存済みの接続情報を使って継続する。
- macOS のモバイル接続 UI からトークン文字列の表示とクリップボードコピーを撤去する。接続情報の供給は QR コード表示だけとし、トークン再発行は残す。
- ADR 0076 は supersede しない。本決定は QR 採用方針を唯一の接続経路として具体化する追加決定である。

## 棄却案

- **QR を主導線にしつつ手動 token・host・port 入力を残す**: 入力ミスと機密情報の露出経路が残り、Tailscale 前提の実運用に対する代替経路にもならない。
- **macOS のトークン表示・コピーだけ残す**: iOS 側の手動入力撤去後は消費先がなく、全権トークンの露出面だけが残る。
- **`explicitHost` を手動設定 UI として復活させる**: 現行 CompositionRoot から到達不能であり、非 Tailscale 通信の公開範囲・認証情報保護を別途設計する必要があるため採らない。

## 結果

- 手動入力ミスを排除し、全権トークンの画面表示・クリップボード経由の漏洩面を削減する。
- 接続設定の適用経路は QR の host・port・token 一括適用に集約される。
- トークンコピー UI の撤去により、macOS `AgentDomain` の `SecurePasteboard` は本番コードからの参照が 0 件となり、定義とテストだけが残る。削除要否はユーザー判断が必要なため、本 run では削除せずフォローアップとする。
